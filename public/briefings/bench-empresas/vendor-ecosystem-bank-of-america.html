6 Os 4 padrões de compra do BofA Brasil — leitura comercial

Decodificar como o BofA Brasil compra é mais importante que listar quem ele compra. Quatro padrões estruturais definem todo o jogo:

1
Padrão
O que o BofA faz
Cloud privada own-built + cloud pública seletiva. Hari Gopalkrishnan (Group CTIO) declarou economia de US$ 2 bi/ano com cloud própria. Workloads críticas (Quartz, risk, regulatory) ficam internas; periferia migra para Azure + AWS.
Implicação para Foursys
Não vender "modernização para cloud pública total". Vender migração seletiva + integração com private cloud BofA. Squad com domínio dos dois mundos (legacy + hybrid).
2
Padrão
O que o BofA faz
Vendor para "data + workflow" sob contrato multi-anos. LSEG partnership é o template — vendor estratégico assina contrato 3-5 anos com escopo amplo + APIs integradas + integração via LSEG Workspace.
Implicação para Foursys
Posicionar Foursys como "capacity partner multi-ano", não como "vendor de squad pontual". Linguagem alinhada ao gosto do banco: MSA Staff Augmentation com cláusula de scale up/down.
3
Padrão
O que o BofA faz
Back-office terceirizado para grandes consultorias. EY, Infosys McCamish, NCB atendem operações administrativas — não squads tech especializadas. Esse anel é fechado para boutiques.
Implicação para Foursys
Não tentar competir aqui. Competir no anel "specialized tech for IB/Markets" — onde nem EY nem Infosys McCamish entram. Pauta: FICC, Quartz, Risk, Markets tech.
4
Padrão
O que o BofA faz
Body shop dedicado por sub-LOB. Vaga ".NET sênior para BofA" via Sercanto mostra: cada LOB (Markets, GTS, IB, Wealth) contrata seu vendor de allocation separadamente. Não há master contract único de TI.
Implicação para Foursys
Entrar LOB por LOB, não tentar contrato master único. Sequência sugerida: GCIBT/GTS (Roberta) → Payments (Geraldo) → EM FICC (Victor) → Fraude (Lusmary) → IB (Leonardo).

9 FAQ de Procurement — perguntas que Cristiane e Jaqueline vão fazer

Antecipar perguntas é metade da venda. Lista de 10 perguntas que aparecem em todo processo de vendor onboarding em banco global, com respostas Foursys prontas:

Por que vocês e não a Wipro, que já é vendor BofA global?
A Wipro é parceira valiosa para escala — mega-projetos com 200+ pessoas. Para squads enxutas e especializadas em IB/Markets/Bank Python — que é onde o ciclo Matera ou IPO 2027 vai exigir velocidade — a Foursys entrega ramp-up 3× mais rápido com squad 100% Brasil. Não somos substituta, somos complementar para escopos onde Wipro tem fricção (timezone Mumbai + retenção 25-35% + velocity em squad pequena).
Vocês já têm cases em banco internacional como BofA?
Sim — temos histórico em IB tier-1 e bancos top-3 BR. Posso enviar 3 referência clientes (com NDA) cobrindo: (i) squad Quartz-like em FICC/Treasury; (ii) modernização Pix/SPB; (iii) AI Pair Programmer em ambiente air-gapped. Tempo médio de ramp-up auditado: 6 semanas vs. 12-18 do GBS Mumbai.
Qual o blended rate e estrutura de TCO?
Posso enviar TCO completo com cenários otimista/base/pessimista. Para squad nearshore BR ramp-up 6 semanas (vs. 12-18 GBS), economia média de 4.000-8.000h por 10 devs/ano. ROI 4-6× sobre o investimento em academia. TCO 30-50% menor que Capgemini/Accenture; 10-20% acima de Stefanini/TCS bruto, mas 30-40% menor em TCO ajustado por ramp-up + retenção.
Vocês têm ISO 27001? E LGPD compliance?
Sim — Foursys possui ISO 27001 + ISO 27701 (dupla rara local), SOC 2 Type II, LGPD compliance auditada, código de conduta supplier alinhado a BofA Global Supplier Code, e data residency 100% Brasil. Estamos prontos para due diligence em 30 dias se houver fast track.
Vocês usam IA? Como protegem o IP do BofA?
Nosso AI Pair Programmer roda dentro do VDI BofA, com privacy-by-design: RAG indexa apenas conteúdo público (Hull, BIS, ISDA, QuantLib) — código BofA nunca entra no índice; LLM com no-training guarantee (Azure OpenAI ou Bedrock); logs auditáveis 100%; red-teaming pré-aprovado pela security do banco. Zero exfiltração de código. Cláusula contratual de IP residency + revogação imediata em caso de incident.
Qual a curva de turnover? Quantos devs vocês perdem por ano?
Turnover Foursys em squads dedicadas: <10% anual (vs. 25-40% típico de body shops globais blended India/Manila). Modelo squad pod + carreira clara + remuneração competitiva BR. Bench disponível para substituição: 2 semanas (vs. 4-6 semanas mercado).
Como vocês escalam de 5 para 50 pessoas se a parceria evoluir?
